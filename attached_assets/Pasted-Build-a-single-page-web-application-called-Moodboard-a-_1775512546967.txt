Build a single-page web application called "Moodboard" — a minimal, aesthetic, infinite pan-able grid for collecting Substack posts, YouTube videos, website links, and photos.

--- CORE LAYOUT ---
The grid starts from the exact center of the viewport and expands outward in all directions as items are added — like a crystalline growth pattern. Items vary in size (small, medium, large) and are placed algorithmically so the grid always looks balanced and fills outward from the center. The layout must feel organic and Pinterest-like but centre-anchored, not top-down.

The grid is infinite canvas. Users pan by clicking and dragging (desktop), using two-finger trackpad scroll (horizontal + vertical), or touch drag on mobile. No scrollbars. Use CSS transform: translate() on a canvas wrapper div to handle panning. Implement inertia/momentum so panning feels smooth. Zoom in/out with pinch gesture (mobile) or Ctrl+scroll (desktop).

--- ITEM TYPES ---
1. SUBSTACK POST: User pastes a Substack article URL. The app fetches Open Graph metadata (og:title, og:description, og:image) via a CORS proxy (use allorigins.win or similar). Display the cover image, post title, and publication name in the card. Clicking the card opens the original URL in a new tab.

2. YOUTUBE VIDEO: User pastes a YouTube URL. Parse the video ID from the URL. Display the YouTube thumbnail (https://img.youtube.com/vi/{VIDEO_ID}/hqdefault.jpg) with a play button overlay. Show the video title fetched via noembed (https://noembed.com/embed?url={URL}). Clicking opens the YouTube URL in a new tab.

3. WEBSITE / LINK: User pastes any URL. Fetch OG metadata via allorigins.win. Show og:image as cover, og:title as title, and the domain name as subtitle. Clicking opens the URL in a new tab.

4. PHOTO UPLOAD: User clicks an upload button and selects an image file from their device. Display it as a full-bleed image card with no text overlay. Clicking opens the image full-screen in a lightbox.

--- CARD DESIGN ---
Cards are minimal and flat. Rounded corners (12px). Cards come in 3 sizes: small (220px wide), medium (320px wide), large (420px wide) — assigned semi-randomly with a bias toward medium. Height is determined by content.

Cover image fills the top portion of the card (aspect ratio 16:9 or 4:3 depending on source). Below the image: title in 14px medium weight, subtitle/domain in 12px muted color. No drop shadows. On hover, a very subtle scale(1.02) transform with smooth transition. The card border becomes slightly more visible on hover.

For YouTube cards, overlay a circular play icon (SVG) centered on the thumbnail.
For photo-only cards, the image fills the entire card with no text.

--- ADD ITEM UI ---
Floating "+" button fixed to the bottom-right corner. Clicking it opens a minimal modal/drawer from the bottom with:
- A URL input field with placeholder "Paste a link — Substack, YouTube, or any website"
- A "Upload Photo" button that triggers a hidden file input
- A "Add" button that processes the input
- Auto-detect the type of link (YouTube if youtube.com or youtu.be, Substack heuristic, else generic website)
- Show a loading skeleton card while metadata is being fetched
- Close on ESC or clicking outside

--- NIGHT MODE ---

Toggle button: a small pill-shaped button in the top-right corner of the screen. In light mode it shows a crescent moon icon (SVG); in dark mode it shows a sun icon (SVG). Smooth transition on toggle (0.25s ease on background-color and color for all elements).

Every hardcoded color (#FFF, #000, rgba black, rgba white) must be replaced with the appropriate CSS variable.

The skeleton loading animation must use a shimmer effect that works in both modes: use a linear-gradient from --bg-skeleton through --skeleton-shine back to --bg-skeleton, animated with background-position shift.

The modal drawer that appears when adding items must have backdrop blur (backdrop-filter: blur(12px)) in both modes. In dark mode the modal background should be semi-transparent (rgba(28,28,26,0.95)) with a subtle top border using --border-modal.

All transitions between light and dark: apply transition: background-color 0.25s ease, color 0.25s ease, border-color 0.25s ease to all major elements via a global CSS rule so the mode switch feels smooth and never jarring.

--- VISUAL STYLE ---
Font: Inter from Google Fonts. The overall feel should be calm, editorial, gallery-like — similar to Are.na or a minimal Notion moodboard. The accent color (#D4A574) stays the same in both modes and is used only for the "+" FAB button and active/focus states.

--- PERSISTENCE ---
Save all items to localStorage so the grid persists on page refresh. Store each item as: { id, type, url, title, subtitle, imageUrl, size, gridPosition, addedAt }. Also persist theme preference in localStorage under "moodboard-theme". On page load, read both keys and apply them before first paint to avoid any flash.

--- EMPTY STATE ---
When the grid is empty, show a centered soft message: "Your moodboard is empty — add a link or photo to begin" in --text-muted, with a subtle dashed border using --empty-state-border around the center zone.

--- TECH STACK ---
Vanilla HTML, CSS, and JavaScript only. Single index.html file. No frameworks, no build tools, no external JS libraries except Inter from Google Fonts. Must work by opening index.html directly in a browser.

--- EXTRA POLISH ---
- Smooth fade + scale-in animation when a new card appears
- "Reset view" button (small pill, top-left, uses --reset-btn-* tokens) re-centers the canvas with smooth animation
- Small item count indicator (e.g. "12 items") in --text-secondary next to the reset button
- On mobile, a brief "Drag to explore" hint in --text-muted that fades out after 3 seconds on first load
- Theme toggle must also update the meta name="theme-color" tag dynamically so the browser chrome matches on mobile (#F9F8F5 for light, #111110 for dark)
